systems philosophy

if i can't explain it, it's probably wrong.

complexity is debt. every abstraction should earn its place. i prefer systems a new engineer can understand in an afternoon. 

// bad: clever
data.reduce((a,b) => ({...a, [b.k]: [...(a[b.k]||[]), b]}), {})

// good: obvious  
const grouped = groupBy(data, 'category')

boring technology wins.

postgres over the new hotness. proven patterns over novel architectures. the best infrastructure is invisible. 

// i like this stack:
// - postgresql (it does more than you think)
// - redis (for the things postgres shouldn't do)
// - typescript (types catch bugs before users do)

the best code is the code you delete.

every line is a liability. i regularly audit what we don't need. unused features are silent complexity. 

// before: 2,400 lines
// after: 800 lines
// same functionality
// easier to change
